# Superseded specifications (read-only)

These sixteen documents are the **numbered specification set** (`SPMS-SUB-001…010`,
`SPMS-FUN-011…016`). As of reconciled set v1 (2026-06-14) they are **superseded** by the canonical
specifications in `../specifications/` and the programme standards in `../standards/`.

They are retained **read-only** for two reasons:

1. **Provenance.** Three of them were promoted into first-class canonical specifications:
   - `SPMS-SUB-006` → `SPMS-BASE-CCB` (Baseline, Version & Change-Control Engine)
   - `SPMS-SUB-008` → `SPMS-INT-EVENT` (Integration & Event Framework)
   - `SPMS-SUB-009` → `SPMS-DATA-STORE` (Data Persistence Layer)
2. **Detail harvesting.** Several were more detailed than their named counterparts. The net-new
   detail to lift into the canonical specs is tracked in `../DETAIL-HARVEST-BACKLOG.md`.

Do not implement, estimate, or review from these files. The authoritative source is the canonical
set listed in `../programme/SPMS-INDEX_specification-register.md`. Where these files conflict with a
canonical specification, the canonical specification governs.

Supersession map: see the register (`SPMS-INDEX`) section 3.
